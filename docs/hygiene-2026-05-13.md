- `npx next lint` would work, but it is **interactive on first run**: it prompts for ESLint config style (Strict / Base / Cancel), installs `eslint-config-next` and `@types/react`, and rewrites `tsconfig.json` (adds `lib`, `allowJs`, `noEmit`, `incremental`, `jsx`, `plugins[next]`, and an `exclude` block).

2. **No, keep the repo lint-free.** Codify the choice in an ADR so this question doesn't get re-asked every sweep.
